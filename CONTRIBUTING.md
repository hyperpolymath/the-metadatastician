# Contributing

## Scope

Contributions should strengthen metadata governance, documentation integrity, and auditable delivery practices for CC-Studio workloads.

## Workflow

1. Fork the repository and create a branch.
2. Make focused changes with clear commit messages.
3. Run local checks:
   - `./scripts/validate-docs.sh`
4. Open a pull request with:
   - problem statement
   - proposed change
   - validation evidence

## Pull Request Expectations

- Keep claims factual and verifiable.
- Avoid placeholder policy language in committed docs.
- Update `CHANGELOG.md` for user-visible changes.
- Link related issues when applicable.

## Review Criteria

PRs are reviewed for:

- metadata and governance correctness
- clarity and maintainability
- security and compliance impact
- traceability of changes

## Questions

Open a GitHub Discussion or issue for design and policy proposals before large changes.
